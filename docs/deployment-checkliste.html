Was dann passiert

Wenn du grünes Licht gibst

  2. Drei Agenten parallel starten: Build-Fix · Postgres-Migration · Storage-Adapter
  3. Schema via prisma db push auf Supabase deployen
  4. Vercel-Projekt via CLI anlegen + GitHub verbinden
  5. ENV-Variablen via vercel env add setzen (alle ~8 Variablen)
  6. git push → Vercel triggert ersten Build automatisch
  7. Verifizieren: Login funktioniert, Customer anlegen geht, Upload klappt
  8. URL melde ich dir zurück
