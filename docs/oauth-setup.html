Was kommt am Ende dabei raus?

Du brauchst sechs Werte in deiner .env-Datei. Wo die liegt: /Users/kurzeja/lisa-crm/app/.env.

# Zoom
ZOOM_CLIENT_ID=abc123…
ZOOM_CLIENT_SECRET=xyz789…
ZOOM_REDIRECT_URL=http://localhost:3006/api/integrations/zoom/callback

# Google Meet (via Calendar API)
GOOGLE_CLIENT_ID=123-abc.apps.googleusercontent.com
GOOGLE_CLIENT_SECRET=GOCSPX-…
GOOGLE_REDIRECT_URL=http://localhost:3006/api/integrations/google/callback
Wichtig: Diese App-Credentials gehören zur CRM-Software, nicht zu deinem Zoom/Google-Account. Du registrierst sie einmal. Wenn du oder andere User später deren eigenen Zoom-Account verbinden, läuft das alles über dieselben App-Credentials — analog wie Calendly genau eine Zoom-App für Millionen User nutzt.
Z

Zoom-App registrieren

≈ 5 Minuten · marketplace.zoom.us
  1. Marketplace öffnen

    Geh auf marketplace.zoom.us und logge dich mit deinem normalen Zoom-Account ein.

  2. Develop → Build App

    Oben rechts „Develop" → „Build App". Dann den Typ „General App" wählen (das ist die moderne Variante von „OAuth App").

  3. App-Name vergeben

    z.B. photosuite-meetings. Wähle „User-managed App" (für nur dich oder Test-User).

  4. OAuth Redirect URL

    Im Tab „Basic Information" → „OAuth Information":

    Setze OAuth Redirect URL und OAuth allow lists auf:

    http://localhost:3006/api/integrations/zoom/callback

    In Production: ersetze localhost:3006 durch deine Domain
  5. Scopes setzen

    Im Tab „Scopes" füge hinzu:

    meeting:write:meeting user:read:user

    (in der UI sind diese mit „/admin" und „/user"-Varianten gemischt — wähle die user-Varianten, nicht die admin)

  6. Test-User hinzufügen

    Im Tab „Test Accounts" → füge deine eigene Zoom-E-Mail hinzu. Nur diese E-Mails können sich verbinden, solange die App nicht „published" ist.

  7. Credentials kopieren

    Im Tab „Basic Information" oben siehst du:

    → Client ID kopieren → das wird ZOOM_CLIENT_ID

    → Client Secret kopieren → das wird ZOOM_CLIENT_SECRET

G

Google Cloud Project + Calendar API

≈ 8 Minuten · console.cloud.google.com
  1. Google Cloud Console öffnen

    Geh auf console.cloud.google.com mit dem Google-Account, dessen Kalender Meetings erstellen soll.

  2. Neues Projekt anlegen

    Oben links: Projekt-Dropdown → „Neues Projekt". Name z.B. photosuite-meetings. Bestätigen.

  3. Calendar API aktivieren

    Linkes Menü → „APIs & Services" → „Library". Such nach „Google Calendar API", öffnen, „Enable".

    Google Meet selbst hat keine separate API — Meetings werden via Calendar Events erstellt
  4. OAuth Consent Screen konfigurieren

    „APIs & Services" → „OAuth consent screen".

    User Type: External wählen.

    App-Name: photosuite. Support-E-Mail: deine. Developer-E-Mail: deine.

    Speichern → weiter.

  5. Scope hinzufügen

    Im Step „Scopes" → „Add or remove scopes" → such nach calendar.events:

    .../auth/calendar.events

    + .../auth/userinfo.email + .../auth/userinfo.profile auswählen.

  6. Test-User hinzufügen

    Step „Test Users" → füge deine eigene Google-E-Mail hinzu (siehe Abschnitt Test-Mode).

  7. OAuth Client ID erstellen

    „APIs & Services" → „Credentials" → „+ Create Credentials" → „OAuth client ID".

    Application type: Web application.

  8. Authorized redirect URI eintragen

    Im Feld „Authorized redirect URIs" → „+ Add URI":

    http://localhost:3006/api/integrations/google/callback

    Speichern.

  9. Credentials kopieren

    Im Modal direkt nach dem Speichern (oder später im Credentials-Tab):

    → Client ID → GOOGLE_CLIENT_ID

    → Client Secret → GOOGLE_CLIENT_SECRET

In .env eintragen

Die Datei app/.env öffnen und am Ende anhängen (oder bestehende Einträge ersetzen).

ZOOM_CLIENT_ID=<dein Zoom Client ID>
ZOOM_CLIENT_SECRET=<dein Zoom Client Secret>
ZOOM_REDIRECT_URL=http://localhost:3006/api/integrations/zoom/callback

GOOGLE_CLIENT_ID=<dein Google Client ID>
GOOGLE_CLIENT_SECRET=<dein Google Client Secret>
GOOGLE_REDIRECT_URL=http://localhost:3006/api/integrations/google/callback
Dev-Server neu starten nach dem Speichern — Next.js lädt .env nur beim Start. Im Terminal: Ctrl+C → npm run dev.

Im CRM verbinden

Sobald die Credentials da sind, läuft alles über die UI.

  1. Öffne Einstellungen → Kalender

    Scrolle zur Section „Online-Meeting-Tools".

  2. Klick „Mit Zoom verbinden"

    Du wirst zu Zoom weitergeleitet, akzeptierst die Berechtigung und landest wieder im CRM mit der grünen Markierung „verbunden mit deiner@email.com".

  3. Klick „Mit Google verbinden"

    Analog. Bei Google bekommst du ggf. einen Warning-Screen wegen Test-Mode — siehe Abschnitt Test-Mode.

  4. Buchungstyp testen

    Geh in „Buchungslinks", öffne einen Online-Meeting-Typ, wähle als Provider z.B. Zoom. Mach eine Test-Buchung über /b/[slug]. Auf der Bestätigungs-Seite siehst du den frisch erstellten Meeting-Link — bei Zoom unter us05web.zoom.us/j/…, bei Google unter meet.google.com/….

Wenn nicht verbunden: Das CRM fällt automatisch auf den persönlichen Meeting-Link zurück (den du im selben Tab eintragen kannst). So funktioniert alles weiter, falls du z.B. die OAuth-Verbindung mal trennst.

Was bedeutet „Test-Mode"?

Sowohl Zoom als auch Google starten dich im „Test-Mode" — das ist genau richtig für dich.

Solange du allein nutzt: Test-Mode reicht

  • Bei Zoom: nur Accounts auf der „Test Accounts"-Liste in der App-Konfig können sich verbinden.
  • Bei Google: nur Test-User auf der „Test Users"-Liste im OAuth Consent Screen.
  • Beim Verbinden bei Google siehst du einen Warning-Screen „Diese App wurde von Google nicht verifiziert" — auf „Erweitert" → „Weiter zu photosuite (unsicher)". Das ist im Test-Mode normal.
  • Für dich allein: nie verifizieren lassen — du bleibst im Test-Mode dauerhaft.

Erst wenn andere User dazukommen: Marketplace-Publish

Wenn du das CRM später für andere Photographinnen anbietest, lässt du die App einmal von Zoom „published" und von Google „verified" — dann können beliebige Accounts connecten. Die Client-ID/Secret in der .env bleiben gleich. Marketplace-Review dauert je nach Provider 3–14 Tage.

Hilfe — es klappt nicht

Die häufigsten Stolpersteine, sortiert nach Wahrscheinlichkeit.

„ZOOM_CLIENT_ID fehlt"

Du hast die .env-Variablen ergänzt, aber den Dev-Server nicht neu gestartet. Next.js lädt .env nur beim Start. Ctrl+C → npm run dev.

„redirect_uri_mismatch"

Die Redirect-URL in deiner OAuth-App-Config bei Zoom/Google entspricht nicht exakt der in .env. Achte auf:

  • Genau http://localhost:3006 (nicht 3000, nicht https)
  • Kein Trailing-Slash am Ende
  • Pfad ist /api/integrations/zoom/callback bzw. /google/callback

Google: „App nicht verifiziert"

Normal im Test-Mode. Klick „Erweitert" → „Weiter zu photosuite (unsicher)". Wenn du nicht in der „Test Users"-Liste bist, kommst du nicht weiter — füge deine E-Mail dort hinzu.

Zoom: „invalid scope"

Du hast die Scopes nicht im richtigen Tab gesetzt. In der App-Konfig: Tab „Scopes" → exakt meeting:write:meeting + user:read:user hinzufügen, dann oben in der Tab-Leiste auf „Save".

Verbunden, aber Meeting wird nicht erstellt

Schau ins Dev-Server-Terminal: dort siehst du [createMeetingForBooking]-Fehlermeldungen mit dem genauen Grund. Häufigste Ursachen:

  • Token abgelaufen + Refresh fehlgeschlagen → in den Einstellungen trennen und neu verbinden.
  • Bei Google: Calendar API nicht aktiviert — in der Cloud Console nachholen.
