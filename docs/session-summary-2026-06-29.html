✅ ENV-Setup — erledigt

Erledigt 1. APP_ENCRYPTION_KEY in Vercel gesetzt

Der vorhandene SECRET_ENCRYPTION_KEY-Wert wurde unter dem korrekten Namen APP_ENCRYPTION_KEY in der Production-Umgebung dupliziert. Stripe-Keys und SMTP-Passwörter sind damit entschlüsselbar.

Beachte: Beim Pullen war im alten Wert ein literales \n am Ende — wie damals bei der DATABASE_URL. Wert wurde vor dem Setzen bereinigt (44 Zeichen base64 = 32 Byte Schlüssel).

Erledigt 2. CRON_SECRET in Vercel gesetzt

Ein 64-Hex-Zeichen-Zufallswert (32 Byte) wurde generiert und als Production-ENV hinterlegt. Vercel-Cron sendet das Secret automatisch im Authorization-Header bei jedem 5-Minuten-Lauf mit.

Verifikation gegen /api/cron/workflows:
✓ Ohne Auth → HTTP 401
✓ Falsches Secret → HTTP 401
✓ Korrektes Secret → HTTP 200, {"ok":true,"picked":0,"done":0,"failed":0}
